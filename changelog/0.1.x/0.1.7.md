# 0.1.7 — 2026-05-30

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.13` → `^0.9.16` — adopts `ctx.enrich`, `ctx.enrich.echo`, and `ctx.enrich.notice` from the enrichment API introduced in 0.9.15.
- **`census_search_variables`** — query echo (`effectiveQuery`), `totalMatches`, dataset/year, and `notice` on empty results now surface in a typed `enrichment` block. Fields `total_matches`, `dataset`, and `year` removed from `output` (values preserved in `enrichment`).
- **`census_query_data`** — `totalRows`, dataset, and year moved from `output` to `enrichment`. Field `total_rows` removed from `output`.
- **`census_list_datasets`** — `totalCount`, optional `filterApplied`, and `notice` on empty results added to `enrichment`. Field `total_count` removed from `output`.
- **`census_list_geographies`** — `dataset`, `year`, and `totalLevels` moved from `output` to `enrichment`. Fields `dataset` and `year` removed from `output`.
- **`census_compare_geographies`** — `totalCount`, `truncated`, `sortVariable`, dataset, year, and `notice` when truncated moved to `enrichment`. Fields `total_count`, `sort_variable`, `dataset`, and `year` removed from `output`; `total_count`/`sort_variable` renamed to camelCase (`totalCount`/`sortVariable`) in `enrichment`.
- **`format()`** — markdown trailers updated across all affected tools to read counts from `result.rows.length` / `result.datasets.length` / `result.variables.length` (own array) rather than the removed output fields.
